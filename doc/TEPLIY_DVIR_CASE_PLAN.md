- **Репозиторій:** `~/Documents/Repositories/teplyi-dvir` (monorepo `ladomyr`, pnpm + turbo)
- **Live:** https://www.ladomyr.kiev.ua/ (200, задеплоєно на Vercel) — title «Приватна лазня на дровах під Києвом — Ладомир»

- **Безпека:** RBAC, MFA, робота з PII, audit log.

## 3. Чесність / доказова база (КРИТИЧНО)

Проєкт свіжий (розробка червень 2026) → **достовірних маркетингових метрик (трафік, виручка, конверсія) ще немає**. Правило проекту: **жодних вигаданих результатів**.

- Proof = **обсяг збудованого + технічна глибина + реальний live-сайт**, як у Atlas/juliaart (де немає traffic-метрик).
- Дозволені «stat-row» цифри (перевірювані з коду): `30+ таблиць БД`, `18+ модулів CRM`, `Next.js 16`, `AI-асистент`, `MFA/RBAC`, `~15 км від Києва`.
- ЗАБОРОНЕНО: вигадані «+X% бронювань», «окупність», кількість клієнтів, відгуки від імені людей без підтвердження.
- **Framing замість вигаданих «до»-метрик (Gemini):** позиціонувати як «створили базову лінію для вимірювання» — впровадили аналітику/систему, від якої рахуватиметься зростання. Зрілий підхід > прикрашання.
- Якщо згодом з'являться реальні GSC/бізнес-дані АБО письмовий дозвіл клієнта на конкретні цифри — додати окремим апдейтом (як робили з AGENTIS).

## 5. Технічні інваріанти (оновлені політики, обов'язково)

- Self-hosted шрифти: inline `@font-face` (Playfair Display / Manrope / JetBrains Mono, Cyrillic subset) — копіювати блок з наявного `/projects/` кейсу.
- **НЕ додавати `<link rel="preload" as="font">`** (урок ace: піднімає FCP на throttled). `font-display:swap`.
- Inline CSS, аналітика в кінці `<body>`, WCAG (skip-link, aria-label, `<main>`, контраст AA), lazy-loading зображень.
- **Унікальна палітра** ✅ (не повторювати Atlas-золото `#C4841D` / juliaart-cream): **домінанта — глибокий ліс `#2E4034`, ember/теракота `#B4552D` лише як АКЦЕНТ, тепла кремова `#F3ECE3` — фон/текст**. Codex: не робити сторінку занадто кремово-коричневою — темний ліс має домінувати, ember дозувати.
- CWV перевірити на проді після деплою (PSI 3 прогони; ціль LCP<2.5s, CLS<0.1).
